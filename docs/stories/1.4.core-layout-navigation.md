### Component Architecture

[Source: architecture.md#Component Organization]

**File Structure:**

```
components/
├── ui/                     # shadcn/ui components (atoms)
├── sections/               # Page sections (organisms)
├── layout/                 # Layout components (templates)
│   ├── Header.tsx
│   ├── Footer.tsx
│   ├── MobileMenu.tsx
│   └── Breadcrumb.tsx (future)
└── forms/                  # Form components
```

**Component Philosophy:**

- **Atomic Design**: layout components are "Templates" level
- **Server Components First**: Use React Server Components by default (no 'use client' unless interactivity needed)
- **Client Components Only When Needed**: useState, useEffect, event handlers require 'use client'

**Naming Conventions:**

- PascalCase for components: `Header.tsx`, `MobileMenu.tsx`
- Export named components, not default exports (easier refactoring)

[Source: architecture.md#Architectural Patterns]

**React Server Components (RSC):**

- Eliminates client-side data fetching waterfalls
- Reduces JavaScript bundle size
- Improves Core Web Vitals (FCP, LCP)

**Decision for Story 1.4:**

- Header and Footer can be Server Components (static content, no state)
- MobileMenu requires 'use client' (useState for open/close, useEffect for event listeners)
- Active link highlighting requires 'use client' (usePathname hook)

### Accessibility Requirements

[Source: front-end-spec.md#Accessibility]

**Target Compliance:** WCAG 2.1 Level AA

**Keyboard Navigation:**

- Tab order follows logical flow (logo → nav items → contact button → hamburger)
- Focus indicators clearly visible (3px outline, #6b8aff)
- Skip to main content link (hidden until focused, allows bypassing nav)
- Dropdown menus operable with Enter/Space keys
- Mobile drawer trap focus (can't tab outside), Escape key closes
- No keyboard traps

**ARIA Labels:**

```html
<!-- Hamburger menu -->
<button aria-label="Ouvrir le menu de navigation" aria-expanded="false">
  <MenuIcon />
</button>

<!-- Close button -->
<button aria-label="Fermer le menu" aria-expanded="true">
  <XIcon />
</button>

<!-- Dropdown -->
<button aria-label="Menu Services" aria-expanded="false" aria-haspopup="true">
  Services <ChevronDownIcon />
</button>
```

**Screen Reader Support:**

- Use semantic HTML: `<header>`, `<nav>`, `<footer>`
- Use `<ul>` and `<li>` for navigation menus
- Add `aria-current="page"` to active link
- Add Skip to main content: `<a href="#main" className="sr-only focus:not-sr-only">Skip to main content</a>`

**Focus Management:**

- Mobile drawer: trap focus within drawer when open
- Restore focus to hamburger button when drawer closes
- Focus indicators: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-primary-light`

**Color Contrast:**

- Gray 900 (#1A1A1A) on White (#FFFFFF): 15.8:1 ✓
- Primary (#3a67ff) on White: 3.1:1 ✓ (large text only, OK for CTA buttons)
- White on Primary: 6.8:1 ✓ (all text sizes)

### State Management

[Source: architecture.md#State Management]

**No Global State Library Needed (Story 1.4):**

Mobile menu open/close state is local to the MobileMenu component:

```tsx
'use client';

import { useState } from 'react';

export function MobileMenu() {
  const [isOpen, setIsOpen] = useState(false);

  // ...
}
```

**Scroll Detection (Header):**

Use `useEffect` to listen for scroll events:

```tsx
'use client';

import { useState, useEffect } from 'react';

export function Header() {
  const [isScrolled, setIsScrolled] = useState(false);

  useEffect(() => {
    const handleScroll = () => {
      setIsScrolled(window.scrollY > 10);
    };

    window.addEventListener('scroll', handleScroll);
    return () => window.removeEventListener('scroll', handleScroll);
  }, []);

  // ...
}
```

**Active Link Detection:**

Use Next.js `usePathname()` hook:

```tsx
'use client';

import { usePathname } from 'next/navigation';

export function NavLink({ href, children }: { href: string; children: React.ReactNode }) {
  const pathname = usePathname();
  const isActive = pathname === href;

  return (
    <a
      href={href}
      className={isActive ? 'border-b-2 border-primary' : ''}
      aria-current={isActive ? 'page' : undefined}
    >
      {children}
    </a>
  );
}
```

### Performance Considerations

[Source: front-end-spec.md#Design Principles]

**Performance is UX:**

Every animation, every image, every line of code must justify its performance cost. Target: every interaction feels instant.

**Header Performance:**

- Use CSS transitions, not JavaScript animations (GPU-accelerated)
- Debounce scroll event listener (throttle to 100ms)
- Avoid re-renders: memoize scroll state, only update when threshold crossed

**Mobile Menu Performance:**

- Use CSS transforms for slide-in animation (translateX)
- Backdrop blur: `backdrop-filter: blur(4px)` - use sparingly, can be performance-intensive on low-end devices
- Lazy-load MobileMenu component (only render when hamburger clicked)

**Image Optimization:**

- Logo: Use SVG (already exists at `public/assets/logo/kbcom_logo_dark.svg`)
- Footer social icons: Use Lucide React icons (SVG, tree-shakeable)

**Bundle Size:**

- lucide-react: Only import needed icons (tree-shaking)
- Header and Footer as Server Components where possible (zero JS)
- MobileMenu as Client Component (interactive state)

### Animation Guidelines

[Source: front-end-spec.md#Animations & Micro-Interactions]

**Animation Principles:**

1. **Purposeful:** Every animation should communicate state change or guide attention
2. **Subtle:** Prefer understated over flashy
3. **Fast:** Duration 150-300ms for UI interactions, max 500ms for page transitions

**Header Transitions:**

```css
transition: all 300ms ease;
```

**Mobile Menu Slide-in:**

```css
transition: transform 300ms ease-out;
transform: translateX(100%); /* closed */
transform: translateX(0); /* open */
```

**Hover Effects:**

```css
/* Nav link hover */
transition: border-color 150ms ease;

/* Button hover */
transition: all 300ms cubic-bezier(0.44, 0, 0.56, 1);
transform: translateY(-2px);
```

**Reduced Motion:**

```tsx
'use client';

import { useEffect, useState } from 'react';

export function useReducedMotion() {
  const [prefersReducedMotion, setPrefersReducedMotion] = useState(false);

  useEffect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    setPrefersReducedMotion(mediaQuery.matches);

    const handleChange = () => setPrefersReducedMotion(mediaQuery.matches);
    mediaQuery.addEventListener('change', handleChange);
    return () => mediaQuery.removeEventListener('change', handleChange);
  }, []);

  return prefersReducedMotion;
}
```
